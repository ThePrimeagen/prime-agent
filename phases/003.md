## Phase 3 workflow
This changeset is all about contracts.  We need to develop the exact interface changes to make this
changeset happen.

* identify the functions that need to be created to implement this feature
* identify the functions that need to change.
** change the function signature, but do not change the body
* if there is module level data, create it now.  Review ArrowTowerSpec or Enemy specs as an example of module level data.
* if you change a signature of an existing function DO NOT DELETE THE BODY
* if you change a signature of an existing function DO NOT EDIT THE BODY

## Phase 3 done criteria

- [ ] IT IS OK TO CHANGE THE SIGNATURE OF A FUNCTION
- [ ] Every new/changed procedure has a stub body only (`unreachable()`).
- [ ] New constants and module-level data are private, zero-value, and behavior-free.
- [ ] No struct/enum/union edits in `objects/` (unless user escalated to Phase 2).
- [ ] No wiring, call sites, or behavioral logic added.
- [ ] DO NOT VERIFY BUILD
- [ ] DO NOT RUN TESTS
- [ ] Move Linear issue to **Phase 3** before Phase 3 work starts.
- [ ] Phase 3 commit pushed and **PR created or updated immediately**.
